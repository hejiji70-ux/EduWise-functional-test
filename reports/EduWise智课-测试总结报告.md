## 三、缺陷清单

### 致命级（2 个）

**Bug #1 — Admin 登录提示信息泄露用户名是否存在**

- **所属模块**：管理员登录
- **重现步骤**：输入不存在的用户名 + 任意密码 → 点击登录 → 提示"用户名错误"
- **预期结果**：应统一提示"用户名或密码错误"，防止暴力枚举
- **实际结果**：单独提示"用户名错误"，暴露了账号是否注册

**Bug #2 — App 学员登录提示信息泄露**

- **所属模块**：学员端注册登录
- **重现步骤**：输入正确手机号 + 错误密码 → 提示"密码错误"
- **预期结果**：应统一提示"用户名或密码错误"
- **实际结果**：单独提示"密码错误"，暴露手机号已注册

### 严重级（5 个）

| 编号 | 模块 | 问题描述 | 预期行为 |
|------|------|----------|----------|
| Bug #3 | 学科分类管理 | 删除父分类时子分类被一并物理删除，无拦截提示 | 应提示"请先删除子分类"或禁止删除 |
| Bug #4 | 学科分类管理 | 新建同名同级分类返回成功，缺少唯一性校验 | 应返回 400 并提示"同级名称已存在" |
| Bug #5 | 课程审核与管理 | 已上架课程可被直接删除 | 应先下架再允许删除，或直接禁止 |
| Bug #6 | 学员端订单支付 | 视频播放退出后重新进入，进度未保存，从头开始 | 应自动保存进度，再次进入从断点续播 |
| Bug #7 | 讲师端课程发布 | 无章节内容的草稿课程可提交至审核 | 应提示"请先添加章节内容" |

### 一般级（2 个）

| 编号 | 模块 | 问题描述 | 预期行为 |
|------|------|----------|----------|
| Bug #8 | 学员端课程浏览 | 搜索无结果时仅显示"共0条"，缺乏友好提示 | 显示"暂无相关课程"引导文案 |
| Bug #9 | 学员端评论 | 未购买付费课程的学员可正常发表评论 | 应限制为购买后才能评论 |

## 四、测试结论与建议

**总体评价**：系统核心功能基本可用，16 个模块中有 8 个模块通过率达 100%，主业务流程运行正常。

**需优先处理的问题**：
1. **安全类（P0）**：Bug #1 和 #2 的登录提示信息问题，建议立即修复，统一改为模糊提示
2. **数据完整性（P1）**：Bug #3 分类级联删除和 Bug #5 上架课程删除问题，可能造成关联数据丢失
3. **业务逻辑补充（P1）**：Bug #7 空内容审核提交、Bug #9 评论权限校验等属于后端校验缺失
